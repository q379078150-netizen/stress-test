# 渠道压力测试工具

一个轻量的 LLM 渠道压力测试面板：支持并发压测、RPM 开环压测（模拟真实用户流量）、Prompt Cache 命中率测试。

## ⚠️ 安全说明

- **本工具不在代码中存储任何 API Key 或渠道 URL**，全部在网页界面手动填写。
- Key/URL 仅保存在你浏览器的 localStorage（本机），不会进入代码仓库。
- 压测报告（`logs/`、`stress-*.json`）含渠道 URL，已被 `.gitignore` 排除，不会上传。

## 启动

```bash
./launch.sh
# 或
node stress-test-server.js
```

然后浏览器打开 http://localhost:3457

局域网其他设备访问：启动日志会打印 `http://<你的局域网IP>:3457`

## 使用

1. 填入你自己的 **Base URL** 和 **API Key**
2. 点「扫描模型」获取可用模型，或手动输入模型名
3. 选择测试模式：
   - **独立请求 (高QPS)**：闭环压测，找渠道并发极限
   - **连续对话 (Cache测试)**：多轮对话，测 Prompt Cache 命中率
   - **RPM 开环 (模拟真实流量)**：按固定速率发射，模拟真实用户负载
     - 勾选「缓存对话模式」：RPM = 用户/分钟，每用户做多轮缓存对话
4. 设置并发/RPM、持续时间，开始压测
5. 实时监控 + 自动生成报告（可导出 PDF/JSON）

## 配置

无需配置文件。所有参数在界面填写，自动记忆到本地浏览器。

## 端口

默认 3457，可用环境变量覆盖：`PORT=8080 node stress-test-server.js`
